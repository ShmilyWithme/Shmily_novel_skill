# 大纲生成模块

## 功能说明

生成故事大纲，包括总大纲、卷大纲和章节梗概。

## 触发词

- `大纲`
- `规划`
- `章节计划`
- `故事结构`
- `生成总大纲`
- `生成第N卷大纲`
- `规划第N章`

## 总大纲生成流程

### 前置条件
- 项目已初始化
- `project.md` 和 `worldbuilding/settings.md` 已存在

### 执行步骤

1. **读取** `project.md` 和 `worldbuilding/settings.md` 获取项目信息
2. **询问**用户关于主线剧情的核心想法（如有）
3. **生成**总大纲 `outline/master-outline.md`，包含：
   - 故事核心冲突（一句话）
   - 三幕式 / 多卷结构概览
   - 每卷的核心事件、转折点、情感走向
   - 高潮-低谷节奏图（文字描述）
   - 预估字数分配
4. **等待确认**后再进入下一阶段

### 输出格式

```markdown
# [小说名称] 总大纲

## 故事核心
- 核心冲突：（一句话概括）
- 核心主题：（故事探讨的核心问题）
- 核心卖点：（吸引读者的独特之处）

## 多卷结构概览
| 卷数 | 主题 | 核心事件 | 字数 | 情感走向 |
|------|------|----------|------|----------|
| 第一卷 | ... | ... | ... | ... |
| 第二卷 | ... | ... | ... | ... |

## 高潮-低谷节奏图
[文字描述或 ASCII 图表]

## 预估字数分配
- 总字数：XXX万字
- 每卷字数：X-X万字
- 每章字数：3000-5000字
```

## 卷大纲生成流程

### 前置条件
- 总大纲已生成并确认

### 执行步骤

1. **读取**总大纲对应卷的内容
2. **生成** `outline/volume-N.md`，包含：
   - 本卷主题和目标
   - 章节规划表（| 章节 | 标题 | 核心事件 | 字数 | 悬念钩子 |）
   - 本卷角色弧线
   - 伏笔埋设 / 回收计划

### 输出格式

```markdown
# 第X卷 [卷名]

## 本卷概述
- 主题：
- 目标：
- 字数：X万字

## 章节规划表
| 章节 | 标题 | 核心事件 | 字数 | 悬念钩子 |
|------|------|----------|------|----------|
| ch001 | ... | ... | ... | ... |
| ch002 | ... | ... | ... | ... |

## 本卷角色弧线
| 角色 | 起点状态 | 终点状态 | 关键转变 |
|------|----------|----------|----------|
| ... | ... | ... | ... |

## 伏笔计划
- 埋设伏笔：
  - F001: [伏笔内容] - 计划在第X章回收
- 回收伏笔：
  - F001: [伏笔内容] - 在第X章埋设
```

## 章节梗概生成流程

### 前置条件
- 卷大纲已生成

### 执行步骤

1. **读取**以下文件：
   - 卷大纲（本卷的章节规划）
   - 前一章的正文（如有，用于衔接）
   - `notes/misc.md`（伏笔追踪，避免重复埋设或遗漏回收）
2. **检查**伏笔状态：
   - 本章是否有计划回收的伏笔
   - 是否需要埋设新伏笔
3. **生成** `outline/chapter-outs/chNNN.md`，包含：
   - 章节标题
   - POV角色
   - 场景列表（每个场景：地点、参与角色、事件、情绪）
   - 本章要推进的主线/支线
   - 章末钩子（悬念点）
   - 预估字数

### 输出格式

```markdown
# 第X章 [章节标题] 梗概

## 基本信息
- 章节：第X章
- 标题：
- POV角色：（视角人物）
- 预估字数：XXXX字

## 场景列表

### 场景1
- 地点：
- 参与角色：
- 事件：
- 情绪基调：

### 场景2
- 地点：
- 参与角色：
- 事件：
- 情绪基调：

## 剧情推进
- 主线推进：（本章对主线的推进作用）
- 支线推进：（本章涉及的支线）

## 伏笔操作
- 埋设伏笔：（本章要埋设的伏笔）
- 回收伏笔：（本章要回收的伏笔）

## 章末钩子
- 悬念点：（本章结尾留下的悬念）
- 目的：（吸引读者继续阅读的原因）
```

## 注意事项

1. 大纲生成采用交互式确认，每一步等用户反馈后再继续
2. 如果用户已有大纲内容，应保留并在此基础上扩展
3. 大纲应参考对应类型的模板（`templates/genre-templates/`）
4. 伏笔要统一编号，便于追踪
