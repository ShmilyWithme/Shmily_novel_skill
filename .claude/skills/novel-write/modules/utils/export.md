# 导出发布模块

## 功能说明

将小说章节合并为可发布的完整文件，去除写作过程中的元数据注释。

## 触发词

- `导出`
- `合并章节`
- `导出全文`
- `发布`
- `导出小说`

## 导出功能

### 1. 全文导出

#### 触发词
- `导出`
- `导出全文`
- `合并章节`

#### 执行流程

1. **扫描章节目录**
   - 读取 `chapters/` 目录下所有 `.md` 文件
   - 按文件名排序（ch001, ch002, ...）

2. **处理章节内容**
   - 去除 `<!-- -->` 格式的元数据注释
   - 去除多余的空行和分隔线
   - 保留正文内容和章节标题

3. **合并输出**
   - 按顺序合并所有章节
   - 添加分隔符（可选）
   - 输出到 `export/{项目名}_全文.md`

4. **生成报告**
   - 总章节数
   - 总字数（中文字符）
   - 输出文件路径

#### 输出格式

```
[导出完成] export/xxx_全文.md
[共 50 章，200000 字]
```

### 2. 按卷导出

#### 触发词
- `导出第1卷`
- `导出第一卷`
- `导出卷1`

#### 执行流程

1. **读取卷大纲**
   - 从 `outline/volume-XX-outline.md` 读取章节范围
   - 确定起始和结束章节号

2. **筛选章节**
   - 只导出指定范围内的章节文件

3. **合并输出**
   - 输出到 `export/{项目名}_第X卷.md`

### 3. 去除元数据

#### 元数据类型

章节文件中可能包含以下元数据：

```markdown
<!-- 字数: 3500 -->
<!-- POV: 林轩 -->
<!-- 场景数: 3 -->
<!-- 推进主线: 林轩觉醒前世记忆 -->
<!-- 埋设伏笔: F001-神秘玉佩 -->
```

#### 处理规则

- 删除所有 `<!-- ... -->` 格式的注释
- 删除 `---` 分隔线（章节之间的分隔由导出格式决定）
- 保留 `#` 标题格式
- 保留正文段落格式

## 注意事项

1. **备份提醒**：导出前建议备份原始章节文件
2. **编码格式**：输出文件使用 UTF-8 编码
3. **文件命名**：使用项目名作为文件名前缀，避免覆盖
4. **目录创建**：如果 `export/` 目录不存在，自动创建
