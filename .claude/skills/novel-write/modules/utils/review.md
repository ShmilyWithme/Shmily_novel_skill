# 审稿润色模块

## 功能说明

提供审稿、润色、查重等文字优化功能。

## 触发词

- `审稿`
- `润色`
- `查重`
- `改写`
- `扩写`

## 审稿功能

### 触发词
- `审稿`
- `审阅`
- `检查章节`

### 审稿流程

1. **读取**最近一章的内容
2. **分析**以下方面：
   - 情节逻辑
   - 人物塑造
   - 文笔质量
   - 节奏控制
   - 悬念设置
3. **输出**审稿报告

### 审稿报告格式

```markdown
# 审稿报告

## 基本信息

- 字数：XXXX字

- 评分：X/10
- 优点：[列出优点]
- 问题：[列出问题]

## 详细分析

### 情节逻辑
- 评分：X/10
- 优点：[具体优点]
- 问题：[具体问题]
- 建议：[修改建议]

### 人物塑造
- 评分：X/10
- 优点：[具体优点]
- 问题：[具体问题]
- 建议：[修改建议]

### 文笔质量
- 评分：X/10
- 优点：[具体优点]
- 问题：[具体问题]
- 建议：[修改建议]

### 节奏控制
- 评分：X/10
- 优点：[具体优点]
- 问题：[具体问题]
- 建议：[修改建议]

### 悬念设置
- 评分：X/10
- 优点：[具体优点]
- 问题：[具体问题]
- 建议：[修改建议]

## 修改建议
1. 建议1：[具体修改内容]
2. 建议2：[具体修改内容]
3. 建议3：[具体修改内容]
```

## 润色功能

### 触发词
- `润色`
- `打磨`
- `优化文字`

### 润色流程

1. **读取**最近一章的内容
2. **分析**可以优化的部分：
   - 语言表达
   - 句式结构
   - 用词准确性
   - 描写生动性
3. **生成**润色版本
4. **对比**原文和润色版本

### 润色要点

**语言表达**
- 消除语病
- 优化句式
- 增强表现力
- 提高流畅度

**用词准确性**
- 替换不准确的词汇
- 避免重复用词
- 使用更生动的词语
- 保持风格一致

**描写生动性**
- 增加细节描写
- 优化感官描写
- 增强画面感
- 提高代入感

### 润色输出格式

```markdown
# 润色报告

## 润色范围
- 章节：第X章
- 润色时间：YYYY-MM-DD HH:MM

## 润色统计
- 原文字数：XXXX字
- 润色后字数：XXXX字
- 修改处数：XX处

## 主要修改

### 修改1
- **位置**：第X段
- **原文**：[原文内容]
- **润色**：[润色内容]
- **原因**：[修改原因]

### 修改2
- **位置**：第X段
- **原文**：[原文内容]
- **润色**：[润色内容]
- **原因**：[修改原因]

## 润色效果
- 语言表达：[提升程度]
- 用词准确性：[提升程度]
- 描写生动性：[提升程度]
- 整体质量：[提升程度]
```

## 查重功能

### 触发词
- `查重`
- `重复检查`
- `用词检查`

### 查重流程

1. **读取**最近一章的内容
2. **分析**以下重复：
   - 重复用词
   - 重复句式
   - 重复描写
   - 重复情节
3. **输出**查重报告

### 查重报告格式

```markdown
# 查重报告

## 查重范围
- 章节：第X章
- 查重时间：YYYY-MM-DD HH:MM

## 查重统计
- 总词汇数：XXXX
- 重复词汇：XX个
- 重复率：XX%

## 重复用词

### 高频词汇
| 词汇 | 出现次数 | 位置 |
|------|----------|------|
| 突然 | 8 | 段落1,3,5,7,9,11,13,15 |
| 微微 | 6 | 段落2,4,6,8,10,12 |
| 不禁 | 5 | 段落1,5,9,13,17 |

### 重复句式
| 句式 | 出现次数 | 位置 |
|------|----------|------|
| "XXX，他/她XXX" | 4 | 段落3,7,11,15 |
| "XXX，心中XXX" | 3 | 段落2,6,10 |

### 重复描写
| 描写类型 | 出现次数 | 位置 |
|----------|----------|------|
| 眼神描写 | 5 | 段落1,4,7,10,13 |
| 心理描写 | 4 | 段落2,5,8,11 |

## 修改建议
1. "突然"：可替换为"忽然"、"猛然"、"骤然"等
2. "微微"：可替换为"轻轻"、"稍稍"、"略微"等
3. "不禁"：可替换为"忍不住"、"情不自禁"、"不由得"等
```

## 改写功能

### 触发词
- `改写第N章`
- `重写第N章`

### 改写流程

1. **读取**原有章节内容
2. **询问**用户修改方向：
   - 加强打斗场面
   - 对话更自然
   - 增加心理描写
   - 节奏更快
   - 其他要求
3. **生成**改写版本
4. **对比**原文和改写版本

### 改写输出格式

```markdown
# 改写报告

## 改写范围
- 章节：第X章
- 改写方向：[用户指定的方向]
- 改写时间：YYYY-MM-DD HH:MM

## 改写统计
- 原文字数：XXXX字
- 改写后字数：XXXX字
- 修改处数：XX处

## 主要修改

### 修改1
- **位置**：第X段
- **原文**：[原文内容]
- **改写**：[改写内容]
- **原因**：[修改原因]

## 改写效果
- 改写方向达成度：[评估]
- 整体质量提升：[评估]
```

## 扩写功能

### 触发词
- `扩写第N章`
- `扩展第N章第M段`

### 扩写流程

1. **读取**指定段落内容
2. **分析**可以扩展的点：
   - 细节描写
   - 心理描写
   - 环境描写
   - 对话扩展
3. **生成**扩展版本
4. **对比**原文和扩展版本

### 扩写输出格式

```markdown
# 扩写报告

## 扩写范围
- 章节：第X章
- 段落：第M段
- 扩写时间：YYYY-MM-DD HH:MM

## 扩写统计
- 原文字数：XX字
- 扩写后字数：XX字
- 增加字数：XX字

## 扩写内容

### 扩写点1
- **位置**：第X句
- **原文**：[原文内容]
- **扩展**：[扩展内容]
- **类型**：[细节/心理/环境/对话]

## 扩写效果
- 描写丰富度：[提升程度]
- 代入感：[提升程度]
- 整体质量：[提升程度]
```

## 注意事项

1. 审稿要客观，不能偏袒
2. 润色要保持原意，不能改变风格
3. 查重要全面，不能遗漏
4. 改写要符合用户要求
5. 扩写要自然，不能生硬
