# 一致性检查模块

## 功能说明

检查小说中的人物、设定、伏笔的一致性。

## 触发词

- `检查一致性`
- `角色一致性检查`
- `设定一致性检查`
- `伏笔检查`

## 检查类型

### 1. 人物一致性检查

#### 检查内容
- 言行是否符合性格设定
- 能力使用是否超出设定范围
- 人物关系发展是否符合弧线规划
- 外貌描写是否有前后矛盾

#### 检查流程
1. 读取所有角色设定卡（`characters/` 目录）
2. 扫描所有已写章节（`chapters/` 目录）
3. 提取角色出场片段
4. 对比设定与实际描写
5. 输出检查报告

#### 检查要点

**性格一致性**
- 角色的言行是否符合核心性格
- 说话特点是否前后一致
- 行为习惯是否保持

**能力一致性**
- 使用的能力是否在设定范围内
- 能力强度是否符合等级
- 是否有未设定的新能力

**关系一致性**
- 人物关系发展是否合理
- 关系变化是否有铺垫
- 关系状态是否更新

**外貌一致性**
- 外貌描写是否前后一致
- 关键特征是否保持
- 是否有矛盾描写

### 2. 设定一致性检查

#### 检查内容
- 力量体系描述是否一致
- 地理环境描写是否符合设定
- 社会结构前后是否呼应
- 历史文化是否自洽

#### 检查流程
1. 读取世界观设定（`worldbuilding/settings.md`）
2. 扫描所有已写章节
3. 提取设定相关描写
4. 对比设定与实际描写
5. 输出检查报告

#### 检查要点

**力量体系**
- 等级划分是否清晰
- 能力描述是否前后一致
- 突破条件是否合理

**地理环境**
- 地点名称是否一致
- 距离关系是否合理
- 环境描写是否符合设定

**社会结构**
- 政治体系是否清晰
- 势力关系是否合理
- 经济体系是否自洽

**历史文化**
- 历史事件是否前后呼应
- 文化习俗是否一致
- 信仰体系是否完整

### 3. 伏笔一致性检查

#### 检查内容
- 已埋设的伏笔是否都有记录
- 伏笔回收是否按计划执行
- 是否有遗漏的伏笔

#### 检查流程
1. 读取伏笔追踪表（`notes/misc.md`）
2. 扫描所有已写章节
3. 提取伏笔相关描写
4. 对比记录与实际描写
5. 输出检查报告

#### 检查要点

**埋设检查**
- 新埋设的伏笔是否已记录
- 伏笔描述是否准确
- 计划回收章节是否合理

**回收检查**
- 计划回收的伏笔是否已回收
- 回收方式是否合理
- 回收效果是否达到预期

**遗漏检查**
- 是否有未记录的伏笔
- 是否有未回收的伏笔
- 是否有矛盾的伏笔

## 检查结果

## 自动检查机制

### 写后自动检查

每次章节写完后，自动执行以下检查：

1. **人物检查**
   - 本章出场角色的言行是否符合设定
   - 是否有新出现的角色需要创建设定

2. **设定检查**
   - 本章的设定描写是否符合世界观
   - 是否有新出现的设定需要记录

3. **伏笔检查**
   - 本章是否埋设了新伏笔
   - 本章是否回收了计划中的伏笔

### 定期全面检查

建议每写完10章或每卷结束后，执行一次全面检查：

1. 扫描所有已写章节
2. 检查所有角色设定
3. 检查所有世界观设定
4. 检查所有伏笔记录

## 注意事项

1. 检查要全面，不能遗漏
2. 报告要详细，便于修改
3. 建议要具体，便于执行
4. 问题要分级，便于优先处理
5. 检查要定期，不能堆积
