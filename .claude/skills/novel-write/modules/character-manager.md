# 人物管理模块

## 功能说明

管理角色设定，包括创建、修改和维护角色设定卡。

## 触发词

- `新建角色`
- `创建角色`
- `角色设定`
- `人物卡`
- `修改角色`
- `更新角色`

## 角色设定卡格式

参考 `templates/character-card-template.md`，每个角色文件包含：

```markdown
# [角色名] 设定卡

## 基本信息
- 姓名：
- 别名/称号：
- 年龄：
- 性别：
- 外貌特征：（关键辨识特征，3-5 个要点）

## 性格画像
- 核心性格：（用 3 个关键词概括）
- MBTI 类型：（可选）
- 说话特点：（口头禅、语气、用词习惯）
- 行为习惯：（标志性小动作）

## 背景故事
- 出身：
- 经历：（关键人生事件，影响性格的部分）
- 当前状态：

## 能力体系
- 战斗能力：
- 特殊能力/技能：
- 弱点/短板：

## 人物关系
| 角色名 | 关系 | 当前状态 | 发展方向 |
|--------|------|----------|----------|
| ...    | ...  | ...      | ...      |

## 角色弧线
- 起点状态：
- 转折事件：
- 终点状态：
- 核心成长主题：

## 写作备忘
- 出场章节：chXXX, chXXX...
- 关键对话语录：（保持风格一致的参考）
- 禁忌：（这个角色绝对不会做的事）
```

## 新建角色流程

### 第一步：收集信息

询问用户以下信息（如果用户未主动提供）：

1. **角色名称**：角色的姓名
2. **角色类型**：主角/配角/反派/龙套
3. **基本设定**：年龄、性别、外貌等
4. **性格特点**：核心性格特征
5. **背景故事**：出身和经历
6. **能力体系**：拥有的能力
7. **人物关系**：与其他角色的关系

### 第二步：创建文件

1. 在 `characters/` 目录下创建角色文件
2. 文件名使用拼音或编号（如 `zhang-san.md` 或 `char-01.md`）
3. 按照模板填写角色信息

### 第三步：更新索引

更新 `characters/index.md`，添加新角色的索引：

```markdown
# 角色索引

| 角色名 | 文件名 | 类型 | 首次出场 | 状态 |
|--------|--------|------|----------|------|
| 张三 | zhang-san.md | 主角 | ch001 | 活跃 |
```

## 修改角色流程

### 第一步：读取现有设定

1. 读取指定角色的设定卡
2. 展示当前设定内容

### 第二步：询问修改内容

询问用户需要修改的部分：
- 基本信息
- 性格画像
- 背景故事
- 能力体系
- 人物关系
- 角色弧线

### 第三步：更新设定

根据用户要求更新角色设定卡

## 角色一致性检查

### 触发词
- `检查角色一致性`
- `角色检查`

### 执行流程

1. **读取**所有角色设定卡
2. **扫描**所有已写章节中该角色的出场
3. **检查**以下项目：
   - 言行是否符合性格设定
   - 能力使用是否超出设定范围
   - 人物关系发展是否符合弧线规划
   - 外貌描写是否有前后矛盾
4. **输出**检查报告，标注问题章节和行号

### 检查报告格式

```markdown
# 角色一致性检查报告

## 检查范围
- 角色：[角色名]
- 检查章节：ch001-ch020

## 检查结果

### 通过项
- [x] 言行符合性格设定
- [x] 能力使用在设定范围内
- [x] 人物关系发展符合弧线规划

### 问题项
- [ ] ch005:15 - 外貌描写与设定不符（设定：黑发，描写：金发）
- [ ] ch012:8 - 使用了未设定的能力（瞬移）

## 建议修改
1. ch005:15 - 将"金发"改为"黑发"
2. ch012:8 - 删除瞬移描写，或补充能力设定
```

## 角色档案管理

### 角色分类
- **主角**：故事的核心人物
- **配角**：重要的辅助角色
- **反派**：与主角对立的角色
- **龙套**：短暂出场的角色

### 角色状态
- **活跃**：正在出场的角色
- **退场**：已退场的角色
- **死亡**：已死亡的角色
- **待定**：尚未出场的角色

## 注意事项

1. 角色文件名使用英文或拼音，避免中文路径问题
2. 角色设定要详细，便于后续写作参考
3. 人物关系要清晰，避免混乱
4. 角色弧线要合理，符合故事发展
5. 定期检查角色一致性，避免设定崩塌
